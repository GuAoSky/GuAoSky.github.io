Maven

发表于 2019-05-28 | 分类于 Java
1. 用于管理几十乃至上百个jar包
Maven将jar包统一管理在jar包仓库里实现代码重用，节省项目空间。
jar包的坐标保存于pom.xml文件中。
2. 将代码即时编译类似Eclipse和IDEA等功能
3. 单元测试一次性执行
4. 打包项目
相较于传统tomcat打包项目，Maven集成tomcat插件只需cd项目文件下运行mvn tomcat:run即可
5. Maven项目相关设置
本地仓库位置：${user.home}/.m2/repository
可在注释下添加自己创建的位置：<localRepository>D:\java_maven\maven_repository</localRepository>
注意：文件路径不要有中文名
6. Maven命令
mvn clean：删除本地编译的所有信息：target文件（个人编译环境可能不同）
mvn compile：对src/main下的java文件进行编译
mvn test：不仅编译src/test下的java文件同时也编译src/main下的java文件
mvn package：打包项目生成war包并同时编译src/main和src/test下的java文件
注意：打包的属性可在pom.xml下更改
mvn install：compile+test+package+将包保存到本地仓库里
7. 本地jar包下载至本地仓库
在windows的cmd命令下，参考下面安装命令安装jar包。
注意：这个命令不能换行，中间用空格来分割的
    安装指定文件到本地仓库命令：mvn install:install-file
    -DgroupId=<groupId>       : 设置项目代码的包名(一般用组织名)
    -DartifactId=<artifactId> : 设置项目名或模块名 
    -Dversion=1.0.0           : 版本号
    -Dpackaging=jar           : 什么类型的文件(jar包)
    -Dfile=<myfile.jar>       : 指定jar文件路径与文件名(同目录只需文件名)
安装mysql-connector-java.jar命令实例：
    mvn install:install-file -DgroupId=mysql -DartifactId=mysql-connector-java -Dversion=8.0.11 -Dpackaging=jar -Dfile=mysql-connector-java-8.0.11.jar
执行完命令后，可看到SUCCESS字样的提示，代表安装成功了，可以在本地仓库找到了。
