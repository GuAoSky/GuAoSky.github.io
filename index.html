    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-jdbc</artifactId>
        <version>2.2.1.RELEASE</version>

    spring.datasource.username=
    spring.datasource.password=
    spring.datasource.url=jdbc:sqlserver://127.0.0.1:1433;DatabaseName=

1.Java中equal与==的区别

1.1 ==：
    当变量为基本数据类型时，比较的是两边值的大小。
    当变量为引用类型时，比较的是其指向的地址值。
1.2 equal():
    当没有对equal()重写时，比较的是指向的地址值。
    当重写了equal()时，比较的是其指向的内容。例如(Double,Date,Integer,String)等包装类
    equal()不能用于基本的数据类型变量。

数据库连接池

发表于 2019-05-28 | 分类于 Java

概念：

用于存放数据库连接的容器（集合）。

意义：

普通jdbc执行close就会释放该资源，下次再调用时就得重新创建对象，这样频繁创建对象，造成浪费资源的同时
也降低执行效率。
节约资源，用户访问高效

实现步骤：

标准接口：DataSource（java.sql）
    方法：
        创建数据库连接池对象：ComboPooledDataSource()
        获取连接对象：getConnection()
        归还连接：Connection.close()
        注意：如果Connection对象是由连接池中获取的，那么执行close()方法则不会释放该资源，而是将该资源返回连接池中。
        交于数据库厂商实现
